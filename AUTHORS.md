# Authors

This file recognizes everyone who has contributed to Mendel. It's separate from `MAINTAINERS.md`, which tracks ongoing decision-making responsibility — this file is a record of contribution, full stop, regardless of how much or how long.

## How you get added

Open a PR adding your own entry (or ask a maintainer to add it) the first time you make a merged contribution — code, docs, eval design, or otherwise. Alphabetical by name is fine; there's no ranking implied by order.

## A note on what counts

Code isn't the only contribution that matters here. Writing an eval, catching a subtle bug in a module's README, improving the roadmap, running an experiment that failed and writing it up in `docs/` so nobody repeats it — all of that belongs on this list. If you're not sure whether your contribution qualifies, it does.
