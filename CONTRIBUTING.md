# Contributing to Mendel

Thanks for your interest! Please follow these steps before submitting a PR:

1. Read the `MISSION.md` to understand project goals.
2. Put changes in the appropriate module directory.
3. Keep changes focused — split files that do more than one job.
4. Add or update an eval in `/eval` for capability claims.
5. Run tests and linters where applicable.
6. Fill the PR template and reference related issues.

See `CODE_OF_CONDUCT.md` for behavior expectations.
